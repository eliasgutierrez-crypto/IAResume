# AI Study Summarizer

A simple web application that lets users convert long pieces of text into concise
bullet-point summaries powered by the Google Gemini language model. It's
intended to help students and professionals quickly digest study material.

---

## 📌 Problem Definition
Reading through lengthy articles or notes can be time-consuming. Users need a
fast way to generate study-ready summaries without leaving the browser. This
project solves that by providing an authenticated service where users submit
text and receive a set of five clear bullet points for review.

---

## 🚀 Features

- User registration and login
- JWT-based authentication for protected routes
- Password hashing with bcrypt for security
- Submit arbitrary text and receive a summary generated by Gemini
- Store and retrieve past summaries per user

---

## 🛠 Technologies Used

| Layer      | Technologies                        |
|------------|-------------------------------------|
| Backend    | Node.js, Express, Mongoose          |
| Database   | MongoDB (hosted via connection URI) |
| Authentication | jsonwebtoken, bcrypt              |
| API Client | axios                              |

## 📁 Installation

1. Clone the repository:
   ```sh
   git clone <repo-url>
   cd IAResume
   ```
2. Install backend dependencies:
   ```sh
   cd backend
   npm install
   ```
3. (Optional) Install frontend tooling if you extend it — none required by
   default.

---

## 🔐 Environment Variables
Create a `.env` file in the `backend` directory with the following keys:

```env
PORT=5000                         # API port (default 5000)
MONGO_URI=<your mongodb uri>      # e.g. mongodb+srv://...
JWT_SECRET=<a secure secret>      # used to sign tokens
GEMINI_API_KEY=<your Gemini key>  # required for summarization
GEMINI_ENDPOINT=<optional url>    # override default endpoint
GEMINI_MODEL=<optional model>     # e.g. gemini-1.0
```

---

## ▶️ Running the Backend

From the `backend` folder:

```sh
# development (restarts on file save)
npm run dev

# production-style
npm start
```

The API is then available at `http://localhost:5000/api` by default.

### Available Endpoints

| Method | Path            | Description                          |
|--------|-----------------|--------------------------------------|
| POST   | `/api/register` | Register a new user                  |
| POST   | `/api/login`    | Authenticate and receive a JWT       |
| POST   | `/api/summarize`| Create a summary (requires JWT)      |
| GET    | `/api/summaries`| List summaries for the logged-in user|

---

## 🖥 Running the Frontend

Open `frontend/index.html` directly in a browser. It communicates with the
backend on the same host (use a proxy or CORS settings if serving from a
different origin).

The page includes forms for login/registration, a text area for input, a
button to trigger summarization, and a list of previous summaries.

## 🌐 Deploying on Render

1. **Create two services** on Render:
   - **Web Service** for the backend
   - **Static Site** (or Web Service) for the frontend if you'd like to
     publish it separately.

2. **Backend service settings**:
   - Connect to the GitHub repo and choose the backend folder as the root.
   - Set the build command to `npm install` and start command to `npm start`.
   - Add the environment variables from above via the Render dashboard.
   - Specify `PORT` (Render provides `10000+`; use `${PORT}` or leave default).

3. **Frontend deployment** (optional):
   - Link the repository, point to the `frontend` directory.
   - No build command required if it's just static HTML.
   - Set the publish directory to `/`.
   - If hosted separately, adjust the `api` variable in the HTML to point to
     your backend URL.

4. **MongoDB** can be hosted with MongoDB Atlas or Render’s managed database
   and the connection string placed in `MONGO_URI`.

5. **Gemini key** must be added to Render’s environment variables as
   `GEMINI_API_KEY`.

Once deployed, Render will build and host both services. Use the provided URL
to interact with your app from anywhere.
